## 4. Lessons (explicit)

1. **Production state is the only authoritative signal.** CLI "success" output, a workflow's green check, and `migration list --linked` are all proxies that lied at least once each this arc. The two-signal rule ‚Äî pair every action-layer claim with a state-layer read (MCP `SELECT` against production) ‚Äî caught all four failures. Keep it.
2. **Tool-installation state is a hidden precondition.** A runbook that says "run `supabase ‚Ä¶`" silently assumes the CLI exists and is on PATH. The single-version diagnostic probe (`migration repair --status reverted <one version>` + immediate MCP re-read) is a cheap, decisive way to prove the CLI reaches the intended target before running a 20-version bulk command.
3. **The production-write boundary held under pressure.** At the frustration peak, MCP `apply_migration` looked like a 5-minute shortcut. Staying on the pipeline path cost ~5 terminal commands and kept the deploy mechanism exercised + the history clean. The principle (agents don't write to production) earned its keep precisely because it was inconvenient.
