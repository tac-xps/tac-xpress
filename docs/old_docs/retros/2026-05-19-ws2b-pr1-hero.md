## 2. The two Group-1 fixes — what shipped

| Defect | Before | After |
|---|---|---|
| AWB form wrapper | `w-full max-w-2xl relative p-1 bg-secondary/5 border border-secondary/20 tac-fui-hover mb-16` | `w-full max-w-2xl relative p-1 bg-card border-2 border-border focus-within:border-primary focus-within:shadow-brutal-sm tac-fui-hover mb-16 transition-colors` |
| AWB input child | unchanged (`placeholder:text-muted-foreground` retained; wrapper now provides shell) | unchanged |
| `GET A QUOTE` button | `size="lg"` + `h-14 ... text-sm px-10` + icon `mr-3 w-5 h-5` | `h-11 ... text-xs px-6` + icon `mr-2 w-4 h-4` (no `size="lg"`) |
| `CONTACT SALES` button | same as GET A QUOTE | same as GET A QUOTE |
| Secondary CTA row gap | `gap-3` | `gap-2` |

### 2.1 Decisions documented

- **Placeholder color kept at `text-muted-foreground`** — the WS-2B plan suggested lifting to `text-foreground/50`. Inspecting after the shell fix, `text-muted-foreground` against `bg-card` already passes WCAG AA and reads as visibly-placeholder. Lifting to `/50` would have made the placeholder approach "content" weight on the new shell — over-emphasizing it. Held at `text-muted-foreground` as the conservative, axe-verified choice. Plan § 5 Group 1 updated to reflect the actual decision.
- **`size="lg"` removed from secondary CTAs** — the Button primitive's `size="lg"` sets a default height that competed with the explicit `h-11`. Dropping the prop and letting `h-11` carry the height is cleaner.
- **`tracking-paper-20` kept on secondary CTAs** — the secondary CTAs use the same tracking the LOCATE button uses for letterform consistency across the hero. Reducing tracking on the secondary CTAs would have introduced two different mono treatments in one frame.
