## 8. Honest read

Three things I'd do differently / be cautious about extrapolating:

**(a) The CodeRabbit-iteration cost was higher than the per-PR retros showed.** PR #123 alone took 4 rounds of substantive findings + 1 declined nitpick = 5 cycles. Each cycle: read finding, fix, push, reply, wait for re-review. Conservatively 10–15 minutes per cycle, so ~hour of round-trip on top of the original test floor. The retros mention this in passing but don't budget for it explicitly. **Sprint 2 prompts should pre-budget ≥2 bot rounds per substantive PR.**

**(b) The "abstract on second use" rule held this arc but has a near-future stress test.** When shipment.service.ts lands (3rd consumer of the inline mockImplementation pattern), extracting `makeBuilderSpy` becomes overdue, not just acceptable. If that PR ships without the extraction because "no third pattern yet," the rule has decayed into "abstract on third use" — which is too late. The right framing: extract at TWO consumers, deferred only when explicit project-discipline reasons (e.g., the audit doc still uses a fundamentally different shape).

**(c) Repo-specific vs generalizable patterns in the catalog:** entries 1, 2, 5 (value-contract, multi-step-path, no-line-numbers) generalize to any TypeScript/Vitest codebase. Entries 3, 4, 6 (statSync.isFile, sweep-describe-block, regex-generalize) are general practices but particular to this repo's sentinel-test discipline — other codebases may have different sentinel patterns where these don't apply. Entries 7, 8, 9 (anchor-windows, satisfies+Exclude, abstract-on-second-use) are universally applicable. **The catalog should NOT be lifted wholesale into another repo** — entries 3/4/6 need adaptation.

The most surprising emergent property: CodeRabbit's bot-side learnings have made the catalog SELF-MAINTAINING. New PRs in this repo inherit the patterns automatically because CodeRabbit's memory references them. The catalog file at [`docs/patterns/coderabbit-catalog.md`](../patterns/coderabbit-catalog.md) is the human-side mirror of that bot-side state. If CodeRabbit's memory is ever wiped or re-seeded, the file is the recovery artifact.
